# Stage 3 Redesign Plan — Statistical correctness, new readouts, runtime configurability

Status: design (no code yet). Scope: `workflow/scripts/03_analyze_library.py`,
`workflow/scripts/common.py`, with contract changes from
`workflow/scripts/02_call_variants.py`. Driven by the statistical/engineering
review and the Q&A that followed.

Design principle running through everything: **every statistical knob is
adjustable at run time (CLI flag + `config.yaml` key + Snakefile passthrough)
with a sensible built-in default.** Defaults must produce a correct, publishable
run with no tuning; flags exist for exploration and sensitivity analysis.

---

## 1. Statistical core changes

### 1.1 Per-variant, coverage- and NC-matched empirical null (resolves Issue 3, Issue 4)

- For each variant with N reads, build the null by drawing **N WT reads with
  replacement** (single-molecule, no PCR → WT molecules are i.i.d. draws from
  the process, so p(1−p)/N variance with no finite-population correction is
  correct; W ≈ 200k ≫ N so with-replacement is also numerically exact).
- Compare the **full variant read set as a single point estimate** to that null
  (the variant is *not* resampled). This removes the with-/without-replacement
  scheme mismatch (Issue 3) entirely.
- Build the null at the **variant's exact N** (not a coarse grid) — dissolves
  the nearest-depth mismatch (Issue 4).
- **NC-match the null to the variant's own nucleosome-count distribution**, not
  to WT's. Keeps all variant reads (no power loss at N≈1–2k) while removing the
  nucleosome confounder. Configurable via `--nc-match-target`
  (`variant` default | `wt` | `none`).
- Replaces the `auto_coverage_grid` + `null_results_by_depth` machinery with a
  per-variant (or per-stratum, see 1.4) null.

### 1.2 Multiplicity / max-statistic fix (resolves Issues 1–2)

- `best_cluster_p` (min cluster p over clusters and bins) must be calibrated
  against the **per-iteration max cluster statistic** under the null, computed
  the same way (across clusters *and* bins per null iteration), not against the
  pooled distribution of all null clusters. The `null_max_cluster_sums`
  quantity is already computed but unused — wire it in correctly and extend it
  across bins.
- **Single cluster-detection rule** used identically for null and observed
  clusters. Today `_detect_clusters_adaptive` (null, intra-iteration spatial
  percentile) and `_detect_clusters_from_mask` (observed, per-position
  cross-iteration percentile) differ — collapse to one function so the null is
  generated by the exact procedure that detects the observed statistic.
- Per-position BH-FDR is retained but documented/labeled as a **diagnostic**,
  not an independent discovery count (Issue 6: spatial autocorrelation).

### 1.3 Number of null subsamples B — runtime knob with adaptive option

- `--n-null-subsamples` (alias keeps existing `--n-null-iterations`),
  **default 10,000** for the reported run (was 2,000). Rationale: with ~1000
  variants and cross-variant FDR at q≈0.05, p-values must resolve to ≈10⁻³;
  B=10k gives ~30% MC error there, B=2k is too grainy at the threshold.
- **Reported run uses one common B for all variants** so p/q are directly
  comparable (heterogeneous B creates an artifactual resolution ceiling in
  volcano/ranked plots and biases tail FDR). This is the default behavior.
- Optional adaptive/two-stage mode for exploration: `--adaptive-null`
  (default off), `--adaptive-screen-B` (default 2,000),
  `--adaptive-refine-B` (default 50,000), `--adaptive-refine-quantile`
  (refine variants whose screen p is within this factor of the FDR boundary).
  When adaptive is on, enforce a common *minimum* B and emit the resolution
  floor for plotting.

### 1.4 Null reuse via stratification (compute lever — OPT-IN, NOT calibrated)

> **STATUS (P1.8 sweep, array 35333322):** stratification is **~2×
> anti-conservative** and worse at high N (NC-representative mismatch:
> the shared null uses the stratum's representative NC, but each
> member's observed Δ uses its own reads → a small fixed bias that
> grows as sampling noise shrinks with N). WT-vs-WT: stratify-OFF
> pooled n=616 → BH FP 0/616, p~Uniform (exactly calibrated);
> stratify-ON n=1540 → P(p≤.05)=0.099, BH FP 6/1540. **Resolution:
> default is now stratify-OFF** (exact FDR, and only ~149 testable
> variants in production so it is affordable). Stratification is an
> opt-in speed mode pending the fix below. **Principled fix (future,
> opt-in):** decouple the cheap per-member exact Option-A reference
> from the expensive shared null resampling — recompute each member's
> `wt_ref(member_nc)` and member-NC-reweight the shared subsamples,
> instead of using the stratum representative.

- Nulls depend only on (N, NC-distribution), not on variant footprint data.
  Cluster variants into strata by N (relative tolerance) and NC-distribution
  distance; compute one null per stratum. Likely ~10–100 strata → ~10–50×
  less compute, statistically near-identical when strata are tight.
- `--null-stratify` (default `on`), `--null-strata-n-tol` (default 0.10,
  i.e. ±10% N), `--null-strata-nc-dist` (max Wasserstein-1 between NC
  histograms within a stratum, **default 0.30 nucleosomes**),
  `--no-null-stratify` to force per-variant nulls.
- Measured (Phase 0 micro-benchmark, LDLR subsample, 123 variants):
  variant-vs-WT NC Wasserstein-1 p50 = 0.65, p99 = 2.0 (WT mean NC = 9.2);
  pairwise variant-variant p50 = 0.90. NC variation is small relative to
  the mean → stratification is effective. Default 0.30 keeps strata well
  below the typical variant-vs-WT (0.65) and pairwise (0.90) spread; the
  calibration harness must confirm stratified ≈ per-variant at this value.
- A pre-run diagnostic reports the realized number of strata and the
  within-stratum NC spread so the choice is auditable.

### 1.5 Reproducibility fix

- Replace `random_seed + hash(vid) % 2**31` (process-salted, non-reproducible)
  with a stable hash (`hashlib`) so `--random-seed` makes serial and parallel
  runs bit-reproducible.

## 2. New per-variant readouts

### 2.1 Nucleosome-count shift (recovers what NC-matching deliberately removes)

- **Signed ΔNC mean** = variant mean NC − WT mean NC (direction + magnitude).
- **Wasserstein-1 distance** between variant and WT NC histograms, with an
  empirical p from WT resampling at the variant's N (with replacement, **not**
  NC-matched — the point is to detect the NC difference). KS was considered
  and rejected (discreteness/ties, weak in tails, no effect size).
- Computed for every variant (compute cost negligible vs the footprint null).
  `--skip-nc-readout` to disable; no tuning knobs needed.
- New summary/HDF5 fields: `nc_mean_variant`, `nc_mean_wt`, `nc_delta`,
  `nc_wasserstein`, `nc_shift_p`, `nc_shift_q` (BH across variants).

### 2.2 Minimum detectable effect (MDE) — power transparency

- Per variant, per position: the smallest |Δ| detectable given that variant's
  N and the local WT occupancy p (≈ z_α·√(p(1−p)/N)). Lets a true "no effect"
  (case C) be distinguished from "underpowered."
- `--mde-alpha` (default 0.05). Emitted as a per-variant track and a scalar
  summary (median MDE over the promoter).

### 2.3 TF binding-motif layer

Definition: a contiguous, **sign-consistent** run of significant occupancy
disruption (default in the TF and sub-TF bins), cross-referenced to the causal
variant, aggregated across the many independent SNVs tiling a site, with the
underlying reference DNA extracted.

- Detect clusters on **signed** Δ (separate loss/gain passes) or post-filter
  for sign homogeneity. Knobs: `--motif-bins` (default `TF,sub_TF`),
  `--motif-direction` (default `loss` | `gain` | `both`),
  `--motif-sign-consistency` (default 0.90),
  `--motif-min-width` (default 5), `--motif-max-width` (default 25).
- **Variant-position cross-check**: record whether/how far each significant
  cluster is from the causal variant position. `--motif-require-variant-overlap`
  (default off at the per-variant level; used as an annotation, and as a hard
  filter in the aggregation step).
- **DNA sequence extraction**: requires the reference promoter sequence in
  Stage 3 (see §4). Emit ref sequence for every significant cluster
  (`abs_start..abs_end`).
- **Cross-variant aggregation → motif calls**: per reference position, count
  distinct variants producing a significant same-direction cluster covering it
  ("disruption-density" track); call motif intervals where density exceeds
  `--motif-density-threshold` (default tuned from data, exposed); emit each
  motif's reference sequence and a per-position/per-base sensitivity profile.
  This is the primary biological deliverable.

### 2.4 Protein co-occupancy dependency module (phase 2 — larger change)

Detects case B (variant disrupts protein 1 and a dependent protein 2 elsewhere)
using the single-molecule joint footprint state.

- Uses the **per-read footprint matrix** (`rd.coverage_matrices[label]`,
  reads × positions) which already exists pre-mean but is currently collapsed
  immediately — add a joint/conditional layer that does not collapse.
- WT conditional model: P(footprint at site 2 | footprint at site 1) vs
  P(footprint at site 2 | not). Test whether a variant's site-2 loss is
  predicted by propagating its site-1 loss through the WT dependency
  structure (cooperative) vs appearing independently.
- Library-level: across the many independent SNVs disrupting site 1, does
  site-2 occupancy consistently drop (multiple-instrument evidence)?
- Knobs: `--enable-co-occupancy` (default **off** in phase 1),
  `--cooccupancy-min-variant-instruments`, `--cooccupancy-distal-min-effect`.

### 2.5 Spurious-second-mutation validation (control for §2.4)

- Requires **raw per-read variant calls (`PR`)** carried into Stage 3 (§4).
- For molecules showing a distal footprint loss, inspect raw calls at the
  distal locus: enrichment of a specific second SNV → contaminating
  double-mutant/haplotype, not cooperativity.
- Barcode-composition check (`BK`/`CS`) for collisions; cross-variant
  replication as the strongest filter.
- Knobs: `--secondary-mutation-screen` (default on when co-occupancy on),
  `--secondary-mutation-max-freq`.

## 3. Engineering / correctness fixes (bundled in)

- **Fix `--no-barcode` crash** in `02_call_variants.py`: `read_data` is a tuple
  but the no-barcode path indexes it as a dict (`rd['variant_tag']`). Real
  runtime `TypeError`.
- **De-duplicate** `test_single_variant` / `_variant_test_worker` (~120 lines,
  ~95% identical) into one function; same for
  `_detect_clusters_adaptive` / `_detect_clusters_from_mask`. This is what let
  the null/observed inconsistency arise; fixing it once is required for §1.2.
- **CLI/doc mismatch**: `02`'s epilog, README, and CLAUDE.md show
  `--promoter-chrom`/`--promoter-region START-END` but the parser requires
  `--promoter-region CHROM:START-END`. Reconcile.
- **Unit tests** (currently none): CIGAR/q2r walker, BH, empirical-p formula,
  NC-matched resampling, barcode neighborhood, cluster detection, motif
  sign-consistency, on tiny synthetic inputs. This is the largest engineering
  gap for a statistical pipeline.
- Performance: pass null arrays to workers via shared memory rather than
  pickling; document RAM (process nulls one at a time, keep only summaries —
  never hold all per-variant nulls; ~48 MB per null at B=10k).

## 4. Pipeline-contract changes (Stage 2 → Stage 3)

Two additions to what flows between stages:

1. **Reference promoter sequence into Stage 3** — needed for motif DNA
   extraction (§2.3). Options: add `--reference` to Stage 3, *or* (preferred,
   cheaper, keeps Stage 3 dependency-light) have Stage 2 write the promoter
   reference sequence into BAM/HDF5 metadata. Decision needed.
2. **Raw per-read variant calls (`PR`) into Stage 3** — needed for
   co-occupancy and the spurious-second-mutation control (§2.4–2.5). Stage 3
   currently reads only `PV`/`VC`. Decision: parse `PR` in `parse_bam`, or
   re-derive from CIGAR in Stage 3.

Both must propagate to `config.yaml` and the `Snakefile`
(`analyze_library_extra`).

## 6. Validation & calibration plan

- **Micro-benchmark** (`tools/phase0_benchmark.py`, DONE Phase 0): measured
  on the LDLR subsample — per-iteration ≈ **2.9 ms** single-core at N=130
  (setup negligible; ~linear in N → ≈30 ms at production N≈1500). Confirms
  stratification is the compute lever: at B=10k, 50 strata ≈ 4 CPU-h vs
  ~80+ CPU-h per-variant at production scale. Set `--null-strata-nc-dist`
  default = 0.30 (see §1.4).
- **Null calibration check**: WT-vs-WT negative control
  (`tools/wt_vs_wt_calibration.py`, DONE Phase 0). **Baseline result on the
  current (pre-fix) code is a hard failure**: 300 pure-WT pseudo-variants
  give best_cluster_p median ≈ 0.023, KS-vs-Uniform ≈ 0.50,
  P(p≤0.05) ≈ 0.55 (~11×), BH q<0.05 flags 151/300 (expect ~0). This
  empirically confirms Issues 1–3 and is the **acceptance test for
  Phase 1**: after the fixes, P(p≤α) must ≈ α and BH must call ~0 under
  the null. It is also the deterministic regression oracle for the
  de-dup refactor (`results/phase0/wt_vs_wt_calib.baseline*.json`).
- **Unit tests** per §3.
- **Sensitivity**: vary B, `nc-match-target`, stratification on/off; confirm
  conclusions are stable.

## 7. Phasing

1. **Phase 0 — DONE.** Correctness fixes (`--no-barcode` tuple crash,
   stable blake2b seeding, CLI/doc reconcile); `tools/phase0_benchmark.py`
   (per-iter ≈2.9 ms, NC spread → strata-nc-dist=0.30);
   `tools/wt_vs_wt_calibration.py` (proved current test ~50% false-positive
   under the null — Issues 1–3 confirmed); de-dup of the variant-test and
   cluster-detection functions, verified byte-identical vs locked
   serial+parallel baselines.
2. **Phase 1** — per-variant NC/coverage-matched null with-replacement (§1.1),
   single cluster rule + max-statistic FDR (§1.2), B knob + stratification
   (§1.3–1.4), NC-shift readout (§2.1), MDE (§2.2). Recompute calibration.
3. **Phase 2 — DONE.** Motif layer + DNA extraction + cross-variant
   aggregation (§2.3). §4(1) resolved: Stage 3 `--reference` FASTA.
   Implemented as a post-hoc annotation/aggregation layer (`common.py`
   `annotate_variant_motifs`, `aggregate_motifs`,
   `load_reference_sequence`, `build_motif_cfg`) that does NOT touch the
   validated Phase 1 hot path; new `motifs` HDF5 group + per-sample
   `_motifs.tsv`; motif fields added to the flat `clusters` table and
   per-variant groups. Dev sanity run: coordinate self-check exact
   (0/31 SNV ref bases mismatched); 0 calls on the 33-variant/200-iter
   dev set (correct — gated on calibrated cross-variant FDR; needs
   production data for real motif calls).
4. **Phase 3** — co-occupancy module + secondary-mutation control
   (§2.4–2.5, requires §4 `PR` decision).
